# Slice 24.1 — Runnable production server bundle — Design

**Branch:** `feat/slice-24.1-prod-bundle`
**Date:** 2026-05-29
**Status:** approved (design), pending implementation plan

## Goal

Make `dist/server.cjs` (produced by `npm run build`, run by `npm start`) actually
boot, and add a CI check that catches a non-runnable bundle. Surfaced during
Slice 24: the daemon had to spawn the server via `tsx` because the production
bundle crashes at load.

## Problem (two latent bugs, both from ESM source bundled to CJS)

1. **`import.meta.url` → `undefined` in the CJS bundle.** esbuild emits
   `var import_meta = {}` for `--format=cjs`, so `import.meta.url` is `undefined`.
   - `server/index.ts`: `fileURLToPath(import.meta.url)` → `__dirname`.
   - `server/domain/mcp/builtin/builtin.store.ts`: `createRequire(import.meta.url)`
     throws `TypeError [ERR_INVALID_ARG_VALUE]` at module load, before `listen()`.
   The build stays green (esbuild only warns), so the crash is invisible until run.

2. **Migrations not copied to `dist/`.** `applyMigrations(db, path.join(__dirname,
   'db', 'migrations'))` reads `server/db/migrations/*.sql` from disk at runtime.
   The build never copies them, so even with bug 1 fixed the bundled server boots
   against an empty schema.

Dev (`tsx server/index.ts`) is unaffected: it runs the TS source as ESM
(`import.meta.url` real, `__dirname` = `server/`, migrations found).

## Design

### 1. Fix `import.meta.url` (esbuild banner + define)

Modify only the `dist/server.cjs` esbuild invocation in the `build` script, adding:

```
--banner:js="const import_meta_url=require('url').pathToFileURL(__filename).href"
--define:import.meta.url=import_meta_url
```

esbuild replaces every `import.meta.url` with the banner-defined identifier; in CJS
`__filename` exists, so the value is the real URL of `dist/server.cjs`. This fixes
both `fileURLToPath(import.meta.url)` (→ `__dirname` = `dist/`) and
`createRequire(import.meta.url)`. No source-code change.

- `cli.cjs` does not use `import.meta` → unchanged.
- `aether-shell.js` is bundled as ESM (`--format=esm`) → native `import.meta`, unchanged.

### 2. Copy migrations into `dist/` (build step)

Append to the `build` script:

```
mkdir -p dist/db && cp -R server/db/migrations dist/db/
```

With `__dirname` = `dist/`, `path.join(__dirname, 'db', 'migrations')` resolves
`dist/db/migrations`. Consistent with how `aether-shell.js` is already emitted under
`dist/server/mcp/builtin/`. `server/db/migrate.ts` is unchanged.

### 3. Verification (smoke script + CI job)

- **`scripts/smoke-prod.mjs`** (new): spawns `node dist/server.cjs` with
  `NODE_ENV=production`, `AETHER_FAKE_PROVIDER=1`, `PORT=<free>`,
  `AETHER_DATA_DIR=<temp>`; polls `GET /api/health` for up to ~30s; exits `0` on a
  200, else prints the captured server output and exits `1`. Cleans up the child and
  the temp data dir. Tests the real production path (also serves the SPA from `dist/`).
- **`npm run smoke:prod`**: runs the script (assumes a prior `npm run build`).
- **CI job `prod-bundle`** in `.github/workflows/ci.yml`: `npm ci` → `npm run build`
  → `node scripts/smoke-prod.mjs`. Runs in parallel with the existing `lint + test`
  job. After it passes once, add `prod-bundle` as a second required status check in
  the `main` branch protection.

### 4. Scope & branching

- Slice 24.1 branches from `main` (independent of the CLI). Touches: `package.json`
  (build + smoke scripts), `scripts/smoke-prod.mjs` (new), `.github/workflows/ci.yml`
  (job), and `docs/superpowers/roadmap.md` (mark 24.1 done).
- **Follow-up (Slice 24 / PR #40, after 24.1 lands in `main`):** flip the daemon's
  `serverEntry` in `cli/runtime.ts` back to `dist/server.cjs` (revert the `tsx`
  workaround) and update its expectation. Explicitly NOT done here, because Slice 24
  is not merged yet.

## Testing

- The smoke script is the bundle's integration test (local + CI).
- `migrate.ts` is unchanged; the copy is build-only.
- Manual check: `npm run build && npm run smoke:prod` → exit 0.

## Out of scope

- Switching the bundle to ESM (`--format=esm`) — riskier with native externals.
- Inlining migrations into the bundle — `migrate.ts` change not justified.
- The daemon `serverEntry` flip (belongs to Slice 24).
