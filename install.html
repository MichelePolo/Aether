Install

Un comando, e sei dentro

Prerequisito: Node.js 22+ — l'installazione scarica un tarball precompilato (niente build sul client); better-sqlite3 scarica un binario nativo precompilato sulla maggior parte delle piattaforme.

$ curl -fsSL https://raw.githubusercontent.com/MichelePolo/Aether/main/scripts/install/install.sh | bash
> powershell -c "irm https://raw.githubusercontent.com/MichelePolo/Aether/main/scripts/install/install.ps1 | iex"
$ npm i -g https://github.com/MichelePolo/Aether/releases/latest/download/aether-core.tgz
$ pnpm add -g https://github.com/MichelePolo/Aether/releases/latest/download/aether-core.tgz
$ bun add -g https://github.com/MichelePolo/Aether/releases/latest/download/aether-core.tgz

curl e PowerShell verificano Node, installano l'ultima release e avviano Aether aprendo il browser. Con npm/pnpm/bun, dopo l'install lancia tu aether daemon start --open.

Primo avvio

Avvia e apri

$ aether daemon start --open   # avvia il server e apre http://localhost:3000
$ aether daemon status         # running/stopped + pid + porta
$ aether daemon stop

Disinstallare

Rimuovere Aether

$ aether daemon stop            # ferma il server, se attivo
$ npm  rm -g aether-core        # pnpm: pnpm rm -g · bun: bun rm -g

Rimosso il pacchetto, il comando aether sparisce. I dati locali (sessioni, DB SQLite, chiavi cifrate) vivono nella cartella data/ da cui hai avviato il daemon — o in $AETHER_DATA_DIR se l'hai impostata. Per cancellarli del tutto: rm -rf data in quella cartella.

Troubleshooting

Se qualcosa non torna

La porta 3000 è già occupata

Aether fa bind sulla variabile PORT (default 3000). Scegline un'altra:

$ PORT=3001 aether daemon start --open
$ aether --port 3001 daemon start --open
# da clone, in dev:
$ PORT=3001 npm run dev

Node assente o < 20

Serve Node.js ≥ 20. Installalo e ri-esegui: winget install OpenJS.NodeJS.LTS (Windows), brew install node (macOS), oppure da nodejs.org. Gli script di install lo verificano e si fermano con un messaggio chiaro.

Errore di build di better-sqlite3

Se non esiste un binario precompilato per la tua piattaforma, viene compilato e servono i build tools: Windows = "Visual Studio Build Tools" (workload C++) + Python; macOS = Xcode Command Line Tools (xcode-select --install); Linux = build-essential + python3.

EACCES / permesso negato sull'install globale

Se il prefix globale di npm è di proprietà di root, l'install -g può fallire con EACCES. Usa un prefix utente (nessun sudo):

$ npm config set prefix ~/.local
# aggiungi ~/.local/bin al PATH, poi ri-esegui l'install

Il browser non si apre (SSH/headless)

L'apertura del browser è best-effort. Se non parte, apri manualmente l'URL stampato: http://localhost:3000 (o la porta che hai scelto).
